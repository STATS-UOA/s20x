## What has been added so far

The `tslm()` function was added as a student-facing wrapper for linear models with optional AR(p) errors.

Example syntax:

```r
fit = tslm(log(passengers) ~ t + month + ar(1), data = airpass.df, time = t)
summary(fit)
plot(fit)
normcheck(fit)
anova(fit)
```

Current design:

- `tslm(y ~ x, data = data.df)` falls back to `lm()`.
- `tslm(y ~ x + ar(1), data = data.df, time = t)` uses `nlme::gls()` with `nlme::corARMA(p = 1, q = 0)`.
- The formula parser currently reserves future error terms such as `ma()`, `arma()`, `arima()`, and `sarima()`, but only `ar(p)` is implemented.
- `time = t` can be supplied as an unquoted or quoted column name.
- If `time` is omitted in an AR model, `tslm()` warns that row order is being used.

## Current `tslm()` methods

Implemented or refined so far:

- `print.tslm()`
- `summary.tslm()` and `print.summary.tslm()`
- `plot.tslm()` with residuals versus fitted, residuals over time, ACF, and QQ plot
- `residuals.tslm()` with `type = "response"`, `"pearson"`, or `"normalized"`
- `normcheck.tslm()` with `residualType = "normalized"` by default
- `anova.tslm()`
- Common model generics: `coef`, `fitted`, `predict`, `formula`, `model.frame`, `vcov`, `nobs`, `logLik`, `AIC`, and `BIC`

## Important diagnostic decision

For AR-error models, diagnostic plots should use normalized residuals by default, because these are the residuals after accounting for the fitted correlation structure. Response residuals may still show autocorrelation and can be requested explicitly.

Current intended usage:

```r
plot(fit, residualType = "normalized")
plot(fit, residualType = "response")
plot(fit, which = "acf")
normcheck(fit, residualType = "normalized")
```

Do not silently trim residuals, fitted values, or time values if their lengths do not align. Length mismatches should throw an explicit internal error so bugs are visible.

## AirPassengers teaching example

The user tested this with `airpass.df`:

```r
data(airpass.df)
airpass.df$month = factor(as.character(airpass.df$month), levels = month.abb)
fit = tslm(log(passengers) ~ t + month + ar(1), data = airpass.df, time = t)
summary(fit)
plot(fit)
normcheck(fit)
anova(fit)
```

Important note: `ordered(month, levels = month.abb)` produced polynomial contrasts (`month.L`, `month.Q`, etc.), which were not ideal for teaching. Use an unordered factor with calendar-ordered levels instead:

```r
airpass.df$month = factor(as.character(airpass.df$month), levels = month.abb)
```

## Stage 7.2 result and problem prompting Stage 7.3

Stage 7.2 added normalized residual diagnostics and `anova.tslm()`.

The user then reported that the single-model ANOVA output for a GLS-backed `tslm` object looked too raw and `nlme`-specific:

```text
Denom. DF: 131 
            numDF   F-value p-value
(Intercept)     1 134117.49  <.0001
t               1    837.30  <.0001
month          11     83.37  <.0001
```

The desired Stage 7.3 change is to make the default single-model AR `anova.tslm()` output more student-facing:

- hide `(Intercept)`
- rename `numDF` to `Df`
- rename `F-value` to `F value`
- rename `p-value` to `Pr(>F)`
- add a standard-looking heading such as `Analysis of Variance Table`
- include the response name
- preserve access to the raw underlying `nlme::anova.gls()` output via `verbose = TRUE`
- keep multi-model comparisons delegated to the underlying fitted objects

Because this is a GLS/Wald-style table, avoid adding fake `Sum Sq` or `Mean Sq` columns unless they are statistically defensible.
